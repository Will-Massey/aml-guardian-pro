## Capstone Ecosystem Integration

### Integration with Proposal Software

```
Proposal Software → AML Guardian API

1. User creates proposal for new client
2. Proposal software checks AML status via API
3. If not in system → Auto-create client + trigger screening
4. Display compliance status in proposal
5. Block proposal send if non-compliant
```

### Integration with Practice Management

```
Practice Management ↔ AML Guardian

Bi-directional sync:
- Client created in PM → Auto-create in AML
- Risk assessment updated → Update PM record
- Document verified in AML → Update PM status
- Review due in AML → Create PM task
```

Single view across all Capstone products:
- **Clients**: 150 total, 3 pending AML
- **Proposals**: 12 pending, 2 blocked (compliance)
- **Documents**: 8 expiring this month
- **Reviews**: 5 CDD reviews due
